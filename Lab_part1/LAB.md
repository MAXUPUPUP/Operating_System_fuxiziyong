# 操作系统配套实验 · 第 1~3 课

一套对应"系统调用 / 进程 / fork-exec-重定向"的动手实验。每个 lab 都是一个独立的小 C 程序,跑一跑、改一改,把机制从"看懂"变成"做过"。

## 环境要求
- Linux（或 macOS / WSL）；本实验在 Ubuntu + gcc 13 上验证通过
- 一个 C 编译器（gcc 或 clang）

## 编译与运行
```bash
make            # 一键编译全部,生成 lab1 ~ lab5
./lab1          # 运行某个 lab
make clean      # 清理可执行文件和 out.txt
```

## Lab 1 — fork:调用一次,返回两次
**文件:** `lab1_fork.c`　**对应:** 第三课

```bash
./lab1
```
**观察:**
- `fork()` 之后,父子两个进程都从同一行往下跑。
- 子进程里 `fork()` 返回 0;父进程里返回子进程的 pid（>0）。
- 最后那句"同一行代码"被两个不同 pid 各执行了一次。

**改改看:**
1. 把父进程里的 `wait(NULL);` 删掉,多跑几次,看父子输出顺序会不会乱（体会"谁先跑不确定"）。
2. 连续写两个 `fork()`,数一数最后总共有几个进程在打印（提示:2 的幂）。

## Lab 2 — 父子进程的内存相互独立
**文件:** `lab2_memory.c`　**对应:** 第二/三课

```bash
./lab2
```
**观察:**
- 父子都从 `counter=100` 出发,子 +1、父 +1000,但**互不影响**。
- 关键:打印出来的 `&counter`（地址）父子**相同**,值却不同 —— 因为那是**虚拟地址**,父子各自映射到独立的物理内存（写时复制的结果）。

**改改看:**
- 把 `counter` 从全局变量改成 `main` 里的局部变量,再跑,结论一样吗？（应当一样,栈上的变量同样是各自独立的）

## Lab 3 — fork + exec:运行一个全新程序
**文件:** `lab3_exec.c`　**对应:** 第三课

```bash
./lab3
```
**观察:**
- 子进程 `exec` 之后"变身"成 `ls`,原来的代码段没了,所以 `exec` 后面那句 `perror` 正常情况不会执行。

**改改看:**

## Lab 4 — 亲手实现 `ls > out.txt`（压轴）
**文件:** `lab4_redirect.c`　**对应:** 我们重点讨论过的重定向机制

```bash
./lab4            # 程序会把 ls 的输出写进 out.txt,再读出来给你看
cat out.txt       # 也可以自己直接看文件
```
**观察 —— 这就是 `>` 的真身:**
1. `fork` 出子进程；
2. 子进程 `open` 文件 → `dup2(fd, 1)` 把标准输出(fd 1)改指向文件 → `close(fd)`；
3. 再 `exec` 成 `ls`；
4. `ls` 照常往 fd 1 写,却全进了 out.txt —— ls 自己毫不知情。

**改改看:**
- 把 `dup2(fd, 1)` 改成 `dup2(fd, 2)`,只重定向"标准错误",看 `ls` 正常输出还会不会进文件（不会,因为正常输出走 fd 1）。
- 试着在重定向前后用 `write(1, "marker\n", 7);` 打个标记,观察它去了哪。

## Lab 5 — 缓冲 vs 系统调用（进阶彩蛋 / 经典陷阱）
**文件:** `lab5_buffer.c`　**对应:** 第一课

```bash
./lab5                 # 输出到终端
./lab5 > /tmp/a.txt    # 输出重定向到文件
cat /tmp/a.txt
```
**观察:**
- `write()` 是直达的系统调用,不带缓冲,内容立刻送出。
- `printf()` 走 stdio,在**用户态**攒一个缓冲区。程序里那句 printf **故意不加 `\n`**,于是它可能还停在缓冲区里没刷出。
- 此时 `fork()`,缓冲区**连同地址空间被复制**给子进程；父子退出时各刷一次 —— 那句 printf 就被打印了**两次**！
- **缓冲策略还跟"输出去哪"有关**:输出到终端通常是"行缓冲"（遇 `\n` 就刷）；输出到文件/管道通常是"全缓冲"（攒满才刷）。所以**重定向到文件时这个双打印陷阱最容易现形**。

**改改看:**
- 在 `fork()` 之前加一句 `fflush(stdout);`,再重定向到文件,双打印就消失了 —— 因为缓冲区在复制前已被刷空。这就是修复办法。
